1. Conceptual metaphor — "The Pipeline Glass"

The UI makes the AIDLC pipeline legible through a pane of glass: you see the flow, you see where it's stuck, you see what just moved. The operator is watching and steering a process, not using a website. Three principles:

Glass is transparent to state — the UI never cosmetically hides what the backend is doing (P5). Glass has depth, not ornament — elevation = attention, not aesthetics; density > delight. Glass focuses — when the operator does one thing, that thing owns the glass (G2, SC1).

9. Motion principles

Motion is state-transition communication, not delight. Stage transitions: 150ms ease, no bounce. Drawer: 200ms slide. Exactly one element pulses at a time — the running stage. Optimistic rollback: instant revert + 200ms warning flash. prefers-reduced-motion disables the pulse and slide.

10. What this deck does NOT lock in

Final pixel values (2.5 Refined Mockups tightens). Component API. Backend endpoints (R5 gaps degraded). Marketing/brand — none, internal tool (NG6).
